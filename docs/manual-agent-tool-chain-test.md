# Agent 工具链手动测试文档

本文从用户视角描述当前 AI 对话与工具链的手动测试方式。

目标：

- 验证 AI 是否能基于当前上下文正确规划工具链
- 验证前端是否会显示“调用 XX 工具”
- 验证工具链执行后，Studio 页面状态是否正确变化
- 验证最终回复是否与工具结果一致

## 1. 测试前提

开始测试前，请确认：

1. 后端与前端都已启动
2. 已配置可用的 OpenAI 兼容模型
3. `videos/<series>/` 下至少有一个 series
4. 该 series 下至少有 2 到 3 个视频
5. 至少有一个视频已经生成过 summary

推荐准备一个测试 series：

- `series-a/video-1`
- `series-a/video-2`
- `series-a/video-3`

推荐数据状态：

- `video-1`：已有概况，无导图
- `video-2`：已有概况和导图
- `video-3`：未生成概况

这样最容易验证工具链分支行为。

## 2. 观察点

每条测试都重点观察以下四类结果：

1. 聊天区是否出现“调用工具：XXX”
2. 右侧工具页是否发生预期切换
3. 是否触发生成、跳转、保存笔记等动作
4. 最终 AI 回复是否和工具执行结果一致

## 3. 系列级工具链测试

## 3.1 总结当前系列

用户操作：

1. 进入某个 series 上下文
2. 不选具体视频，停留在系列首页
3. 在聊天框输入：

```text
总结当前系列
```

预期工具链：

1. `调用工具：读取系列视频列表（<系列名>）`
2. `调用工具：读取视频概况（<视频1>）`
3. `调用工具：读取视频概况（<视频2>）`

预期结果：

- 聊天区先出现若干条“调用工具：XXX”
- 不应自动跳到视频预览页
- 不应自动切到笔记页
- 最终回复应是系列级总结，而不是只总结当前某一个视频
- 如果某个视频没有 summary，回复里可以自然提到“部分视频尚未生成概况”

失败信号：

- 完全没有“调用工具：读取系列视频列表”
- 最终回答只围绕一个视频
- 回复编造了不存在的视频内容

## 3.2 询问系列主题脉络

用户输入：

```text
这个系列主要讲了哪些主题
```

预期工具链：

- 与“总结当前系列”类似
- 至少应出现系列视频列表读取
- 常见情况下会读取多个视频概况

预期结果：

- 最终回答应体现“跨多个视频的主题归纳”
- 不应只回复“请打开某个视频查看”

## 4. 视频内定位测试

## 4.1 查询某个知识点出现的位置

用户操作：

1. 进入某个具体视频
2. 在聊天框输入：

```text
API Key 在视频什么位置提到的
```

预期工具链：

1. `调用工具：检索转写（API Key 在视频什么位置提到的）`
2. `调用工具：跳转视频到 MM:SS`

预期结果：

- 右侧工具页应切到“视频预览”
- 视频应跳到对应时间点
- 最终回复应说明大概时间位置
- 回复中不应出现内部术语如 `tool_calls`、`payload`

失败信号：

- 只回答时间，不切预览页
- 切了页但没跳时间
- 没有任何“调用工具：检索转写”

## 4.2 在时间点明确时直接跳转

用户输入：

```text
跳到 2 分 10 秒
```

预期工具链：

- 可能直接出现：
  - `调用工具：跳转视频到 02:10`

预期结果：

- 工具页切到视频预览
- 视频跳到 130 秒附近

## 5. 打开工具页测试

## 5.1 打开知识卡片

用户输入：

```text
打开知识卡片
```

预期工具链：

1. `调用工具：打开知识卡片`

预期结果：

- 右侧工具页切到“知识卡片”
- 如果知识卡片已生成，应看到内容
- 最终回复应自然告知已经打开知识卡片

## 5.2 打开笔记

用户输入：

```text
打开笔记
```

预期工具链：

1. `调用工具：打开笔记`

预期结果：

- 右侧切到笔记页
- 笔记列表正常展示

## 6. 生成 + 打开链路测试

## 6.1 导图未生成时请求打开导图

前提：

- 当前视频 summary 已存在
- 当前视频 mindmap 尚未生成

用户输入：

```text
打开思维导图
```

预期工具链：

1. `调用工具：生成思维导图`
2. `调用工具：打开思维导图`

预期结果：

- 前端应开始导图生成流程
- 生成完成后，右侧停留在思维导图页
- 最终回复应告知正在生成或已经打开

失败信号：

- 只切页不生成
- 只生成不切页
- 没有“调用工具：生成思维导图”

## 6.2 概况未生成时请求打开概况

前提：

- 当前视频尚未生成概况

用户输入：

```text
打开概况
```

预期工具链：

1. `调用工具：生成 AI 概况`
2. `调用工具：打开 AI 概况`

预期结果：

- 应触发 summary 生成流程
- 生成完成后概况页可见

## 7. 笔记工具链测试

## 7.1 让 AI 帮我记笔记

用户输入：

```text
帮我记一下这个视频的重点
```

预期工具链：

1. 可能先做信息检索或直接生成笔记内容
2. `调用工具：保存笔记（<笔记标题>）`
3. `调用工具：打开笔记`

预期结果：

- 右侧切到笔记页
- 新笔记出现在列表中
- 笔记来源应为 `agent`
- 最终回复应说明已经记好

失败信号：

- 回复说记了，但笔记页没有新笔记
- 笔记创建成功但没有切到笔记页

## 7.2 指定内容记笔记

用户输入：

```text
把刚刚关于 API Key 的部分记成笔记
```

预期工具链：

- 若上下文足够，可能直接保存笔记
- 若上下文不足，可能先检索转写，再保存笔记

预期结果：

- 新笔记内容与当前对话主题一致
- 不应保存一条空标题或空内容笔记

## 8. 工具状态判断测试

## 8.1 询问某视频是否已有导图

用户输入：

```text
这个视频有导图吗
```

预期工具链：

1. `调用工具：读取视频工具状态（<video_id>）`

预期结果：

- 最终回答应基于真实工具状态
- 如果已生成，应说已存在
- 如果未生成，应说当前未生成，可继续帮你生成

## 8.2 询问某视频有哪些工具可用

用户输入：

```text
这个视频现在可以看哪些内容
```

预期工具链：

1. `调用工具：读取视频工具状态（<video_id>）`

预期结果：

- 回答应包含概况、导图、知识卡片、笔记、视频预览等可用状态

## 9. 可见性测试

这一组不关注业务结果，只看工具链展示是否清楚。

用户任选以下任一问题：

- `总结当前系列`
- `打开知识卡片`
- `帮我记一下这个视频重点`
- `这个知识点在视频哪里`

预期结果：

- 每当 Agent 真正执行一步工具时，聊天区都应新增一条消息
- 消息格式应类似：
  - `调用工具：读取系列视频列表（Series A）`
  - `调用工具：读取视频概况（Video 1）`
  - `调用工具：打开知识卡片`
  - `调用工具：保存笔记（重点）`
- 工具消息应出现在最终 AI 回复之前

失败信号：

- 工具实际执行了，但聊天区完全不可见
- 只显示最终结果，不显示中间调用

## 10. 异常测试

## 10.1 在 library 级直接问视频内定位问题

用户输入：

```text
这个知识点在视频哪个时间点
```

预期结果：

- 如果当前没有视频上下文，AI 应提示需要先进入某个视频或定位到更具体范围
- 不应盲目返回一个伪造时间点

## 10.2 请求与工作台无关的内容

用户输入：

```text
帮我写一份旅游攻略
```

预期结果：

- AI 应礼貌说明超出当前工作台支持范围
- 不应启动任何工作区工具链

## 11. 验收标准

本轮功能可视为通过的最低标准：

1. 系列级问题能触发多步信息工具链
2. 视频定位问题能触发检索与跳转链路
3. 打开工具页问题能触发对应 UI 工具动作
4. 生成后打开类请求能形成受控双步链
5. 帮记笔记能真正落到笔记列表
6. 聊天区能明确显示“调用工具：XXX”
7. 最终回答与工具结果一致，不编造信息

## 12. 推荐测试顺序

建议按下面顺序手测：

1. `打开知识卡片`
2. `API Key 在视频什么位置提到的`
3. `帮我记一下这个视频重点`
4. `打开思维导图`
5. `总结当前系列`

这样能从最简单的单步动作逐步走到多步链路。
