# Agent 用户模拟测试文档

本文从真实用户视角出发，给出一组可直接照着操作的手动测试用例，用来验证当前 Studio 中的 Agent 对话、事件流、工具链与最终回答是否符合预期。

本文重点验证：

- `思路块` 是否出现且内容合理
- `工具链块` 是否按顺序展示真实工具名
- `最终回答` 是否与工具结果一致
- 系列级问题是否真的走了跨视频工具链，而不是模型硬猜

## 1. 测试前提

开始前请确认：

1. 前端与后端都已启动
2. 已配置可用的 OpenAI 兼容模型
3. 至少存在一个系列，例如 `Agent Frameworks`
4. 该系列下至少有 2 到 4 个视频
5. 至少 1 个视频已经生成过 `summary`
6. 最好同时准备：
   - 一个已有 summary 的视频
   - 一个没有 summary 的视频

推荐样例数据：

- `1-4 准备工作：百度地图API秘钥(AK)`：有 summary
- `1-5 准备工作：安装Nacos 3`：有 summary
- `1-6 仿Manus能自主决策的框架：Jmanus`：有 summary
- `1-7 具备ReAct核心能力的框架：AgentScope`：无 summary 或待生成

## 2. 统一观察点

每条测试都至少观察下面四项：

1. 是否出现 `思路` 折叠块
2. 是否出现 `工具链` 折叠块
3. 展开后是否能看到真实工具名，例如 `list_series_videos`
4. 最终回答是否只引用了真实工具结果，没有幻觉参数或内部技术词

额外关注：

- 不应再出现 `__FROM_LIST__`、`*pending_from_*` 这类伪参数
- 不应把 Planner 的内部理由直接挂在最终回答下面
- 最终回答应显示耗时

## 3. 系列级模拟测试

### 3.1 询问系列主题

用户场景：

- 进入某个系列首页
- 不选具体视频
- 在聊天框输入：

```text
这个系列主要讲了哪些主题？
```

预期思路：

- Agent 应识别为 `series` 级问题
- 不应只围绕当前某一个视频回答
- 如果信息不足，应先读取系列视频列表，再按需读取若干视频摘要

预期事件流：

1. 出现 `正在思考`
2. 思路完成后，显示系列级归纳意图
3. 出现工具链块
4. 展开工具链，至少应看到：

```text
list_series_videos
```

常见情况下还应看到：

```text
get_video_summary
```

预期回答：

- 回答应是系列级主题归纳
- 应能自然提到“准备工作”“框架介绍”“自主决策”“ReAct”等主题
- 如果某个视频没有摘要，允许自然说明“部分视频还没有可用摘要”

失败信号：

- 工具链里没有 `list_series_videos`
- 工具链里出现假参数，如 `__FROM_LIST__`
- 回答只总结了一个视频
- 回答里出现 `tool_calls`、`payload`、`schema` 等内部术语

### 3.2 询问系列学习路径

用户输入：

```text
这个系列适合怎么学？
```

预期事件流：

- 应先出现思路块
- 常见工具链：
  - `list_series_videos`
  - `get_video_summary` 若干次

预期回答：

- 应输出一条学习顺序建议
- 不应只复制视频标题
- 如果只有标题没有摘要，允许回答退回到“基于标题的初步学习路径”

失败信号：

- 完全不使用任何系列级工具
- 回答伪装成知道每期详细内容，但实际没有摘要支撑

### 3.3 询问系列里某两个视频的关系

用户输入：

```text
Jmanus 和 AgentScope 在这个系列里分别承担什么角色？
```

预期事件流：

- `list_series_videos`
- 如有可用摘要，按需 `get_video_summary`

预期回答：

- 应体现“系列内部角色对比”
- 应能区分：
  - `Jmanus` 更偏自主决策
  - `AgentScope` 更偏 ReAct

失败信号：

- 回答把两个视频混成一回事
- 明显编造未从标题或摘要得出的细节

## 4. 视频级模拟测试

### 4.1 询问某知识点在视频哪里出现

用户场景：

- 进入某个具体视频
- 输入：

```text
百度地图 API Key 在视频什么位置提到的？
```

预期事件流：

1. `思路块`
2. `工具链块`
3. 展开工具链应看到：

```text
transcript_lookup
video_seek
```

预期界面行为：

- 右侧切到视频预览
- 预览自动跳到对应时间点

预期回答：

- 自然说明大概时间位置
- 不应只跳转不解释

失败信号：

- 没有 `transcript_lookup`
- 视频没有跳转
- 时间点和回答不一致

### 4.2 直接请求打开知识卡片

用户输入：

```text
打开知识卡片
```

预期事件流：

- 应出现思路块
- 工具链里应看到：

```text
open_knowledge_cards
```

预期界面行为：

- 右侧切到知识卡片页

预期回答：

- 自然说明已经打开知识卡片

### 4.3 请求帮我记重点

用户输入：

```text
帮我记一下这个视频的重点
```

预期事件流：

- 工具链中应看到：

```text
save_note
```

如果实现上会顺手切页，也可能看到：

```text
open_notes
```

预期界面行为：

- 新笔记落到笔记页数据里
- 笔记来源应是 `agent`

预期回答：

- 应明确说“已经帮你记下”

失败信号：

- 回复说记好了，但笔记列表没有新增
- 新增笔记为空标题或空内容

## 5. 生成类模拟测试

### 5.1 没有导图时请求打开导图

前提：

- 当前视频已有 summary
- 当前视频没有 mindmap

用户输入：

```text
打开思维导图
```

预期事件流：

- 工具链应看到：

```text
generate_mindmap
open_mindmap
```

预期界面行为：

- 前端触发导图生成
- 完成后停留在思维导图页

失败信号：

- 只打开，不生成
- 只生成，不打开

### 5.2 没有概况时请求打开概况

前提：

- 当前视频没有 summary

用户输入：

```text
打开概况
```

预期事件流：

- 工具链应看到：

```text
generate_overview
open_overview
```

预期界面行为：

- 触发 summary 生成
- 成功后停留在概况页

## 6. 事件流交互专项检查

这组测试专门看“显示方式是否像一个真正的事件流”。

### 6.1 思路块显示检查

任选一个问题，例如：

```text
这个系列主要讲了哪些主题？
```

检查点：

1. 是否先显示 `正在思考`
2. 思考完成后是否收敛成一个可折叠思路块
3. 思路块是否带耗时
4. 思路块内容是否是“对用户可见的思路摘要”，而不是内部技术碎片

失败信号：

- 思路内容直接暴露内部规划术语
- 思路块根本不出现
- 思路完成后又被最终回答覆盖掉

### 6.2 工具链顺序检查

仍以系列问题为例，展开工具链后检查：

1. 步骤是否有先后顺序
2. 是否显示真实工具名
3. 是否显示状态：
   - `进行中`
   - `已完成`
4. 是否显示每步耗时

失败信号：

- 只有中文说明，没有真实工具名
- 步骤顺序颠倒
- 结果块显示的不是执行后的真实参数

### 6.3 最终回答流式输出检查

用户输入：

```text
总结当前系列
```

检查点：

1. 回答是否逐步出现，而不是整段瞬间出现
2. 回答完成后是否显示耗时
3. 最终回答是否晚于思路块与工具链块出现

失败信号：

- 最终回答比工具链先出现
- 回答一次性整段弹出

## 7. 异常场景模拟测试

### 7.1 在 library 级提视频内定位问题

用户输入：

```text
这个知识点在视频哪个时间点？
```

预期：

- 应提示当前缺少更具体的视频上下文
- 不应直接虚构一个时间点

### 7.2 提出与当前知识库无关的问题

用户输入：

```text
帮我写一份旅游攻略
```

预期：

- 应礼貌拒绝或说明超出当前工作台范围
- 不应调用工作区工具

## 8. 推荐手测顺序

建议按下面顺序手测，最容易定位问题：

1. `打开知识卡片`
2. `帮我记一下这个视频的重点`
3. `百度地图 API Key 在视频什么位置提到的？`
4. `这个系列主要讲了哪些主题？`
5. `这个系列适合怎么学？`
6. `Jmanus 和 AgentScope 在这个系列里分别承担什么角色？`

## 9. 通过标准

本轮 Agent 工具链体验可以视为通过的最低标准：

1. 系列级问题会触发系列级工具链，而不是只猜标题
2. 工具链展示真实工具名，而不是只有中文标签
3. 不再出现 `__FROM_LIST__`、`*pending_from_*` 之类伪参数
4. 思路、工具链、回答三段顺序正确
5. 最终回答与工具结果一致，不编造不存在的摘要内容
