# Agent 主观测试矩阵

本文档总结当前主观测试中需要持续观察的两组用例：

- 核心 Case：聚焦产品级问答、定位、笔记与范围判断的稳定性
- 动作矩阵：覆盖 `open` / `generate` 等动作型工具的调用轨迹

对象测试不做 assert，而是围绕“是否按预期调用工具”“是否自然回答”“是否守住视频范围”来打分。

## 核心 Case

每个 Case 都明确记录“问题”“主要依赖工具”“主要观察点”，方便持续手工/仿真回归时定位表现变化。

### 1. 系列关系总结

- 问题：`百度地图 API Key 这一节和 Nacos 3 这一节在整个课程里分别承担什么作用，它们之间是什么关系？`
- 主要覆盖工具：`list_series_videos`、`get_video_summary`
- 主要观察点：能否在系列层面梳理业务角色、聚合多个视频的要点、避开 transcript 噪音，并真正输出“关系”而非逐个复述。

### 2. 系列概念定位

- 问题：`这个系列里哪里讲过 Nacos 3？最好指出是哪一节，如果能的话给个大概位置。`
- 主要覆盖工具：`list_series_videos`、`get_video_summary`、`get_video_transcript`（必要时带 `video_seek`）
- 主要观察点：能否从系列级问题逐层收敛到某个视频，并沿 summary→transcript 形成证据链，最终准确定位到具体章节。

### 3. 视频原话 + 定位

- 问题：`视频里哪里提到了 0.0.0.0/0？那一段主要在讲什么？`
- 主要覆盖工具：`get_video_transcript`、`video_seek`
- 主要观察点：检索 transcript 的能力、精确定位时戳的能力，以及“位置 + 简述”是否能够同步完成。

### 4. 视频工具状态

- 问题：`这个视频现在有哪些工具已经生成了？导图、知识卡片、笔记分别是什么状态？`
- 主要覆盖工具：`get_video_tools`
- 主要观察点：工具状态读取是否稳定、能否译成用户易懂的话、是否泄漏内部结构与字段。

### 5. 笔记工作流

- 问题：`帮我把这节 API Key 视频的重点记到笔记里，然后把笔记打开。`
- 主要覆盖工具：`get_video_summary`→`save_note`→`open_notes`
- 主要观察点：读取内容与动作串联能力、`save_note` 参数质量、完成动作后回答是否自然。

### 6. video scope 边界

- 问题：`当前这个视频讲的是 API Key，那后面哪一节最可能和它衔接？`
- 主要覆盖工具：当前视频 summary 或保守使用 `direct_response`
- 主要观察点：是否在 video scope 内保持保守答复、不随意越权到整个系列，并确保回答不过于僵硬。

## 动作矩阵

动作矩阵分别对每个动作工具发起短 prompt，验证动作是否能在当前上下文下正常执行。建议一组跑完，观察顺序与失败特征。

| 动作 | 示例 Prompt | 主要覆盖工具 | 补充说明 |
| --- | --- | --- | --- |
| 打开系列首页 | `打开系列首页` | `open_series_home` | 观察系列级动作是否触发、是否仅返回必要 metadata |
| 打开系列概览 | `打开系列概览` | `open_series_overview` | 观察是否能正确选中系列、避免 fallback 复述 |
| 打开概况 | `打开概况` | `open_overview` | 若概况缺失是否联动 `generate_overview`，避免多余问答 |
| 打开思维导图 | `打开思维导图` | `open_mindmap` | 若导图缺失是否衔接 `generate_mindmap`，不走绕路式问答 |
| 打开知识卡片 | `打开知识卡片` | `open_knowledge_cards` | 确保只请求已有卡片，不触发全量生成 |
| 打开视频 | `打开视频` | `open_video` | 观察是否只唤起观看体验，而非过度回答内容 |
| 生成概况 | `生成概况` | `generate_overview` | 关注是否生成结构化概况、是否用 summary 作为素材 |
| 生成导图 | `生成导图` | `generate_mindmap` | 关注导图生成是否抓住知识点与层级结构 |

## 建议人工观察点

- 是否调用了不该调用的工具（过多 fallback、工具链穿透）
- 是否出现空回答、半截回答或重复回答
- 是否泄漏内部字段，如 `tool_name`、`payload`、`selected_tool`
- 复杂问题下是否只是复述多个点，不聚合成明确答复
- video scope 下是否越界胡答其他视频或跨系列
- 动作类请求是否直接触发动作，不被误导绕成问答
