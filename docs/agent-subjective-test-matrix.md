# Agent 主观测试矩阵

本文档记录当前手工/仿真回归里最值得持续观察的两类用例：

- 核心 Case：用于观察产品级问答、定位、笔记工作流和边界处理
- 动作矩阵：用于观察 open / generate 一类动作工具是否正常执行

这些测试不做 assert，重点是：

- 观察事件顺序是否合理
- 观察工具轨迹是否符合直觉
- 观察最终回答是否自然、无内部实现泄漏
- 观察复杂问题是否稳定收口

## 核心 Case

### 1. 系列关系总结

- 问题：
  - `百度地图 API Key 这一节和 Nacos 3 这一节在整个课程里分别承担什么作用，它们之间是什么关系？`
- 主要覆盖：
  - `list_series_videos`
  - `get_video_summary`
- 主要观察：
  - 是否能做系列级关系总结
  - 是否能聚合多个视频 summary
  - 是否错误进入 transcript
  - 是否真正回答“关系”，而不是分别复述

### 2. 系列概念定位

- 问题：
  - `这个系列里哪里讲过 Nacos 3？最好指出是哪一节，如果能的话给个大概位置。`
- 主要覆盖：
  - `list_series_videos`
  - `get_video_summary`
  - `get_video_transcript`
- 可能覆盖：
  - `video_seek`
- 主要观察：
  - 是否能从系列问题收敛到具体视频
  - 是否能走出 summary -> transcript 的逐层证据路径
  - 是否能定位到具体视频

### 3. 视频原话 + 定位

- 问题：
  - `视频里哪里提到了 0.0.0.0/0？那一段主要在讲什么？`
- 主要覆盖：
  - `get_video_transcript`
  - `video_seek`
- 主要观察：
  - transcript 检索能力
  - 定位能力
  - 是否能同时完成“位置 + 解释”

### 4. 视频工具状态

- 问题：
  - `这个视频现在有哪些工具已经生成了？导图、知识卡片、笔记分别是什么状态？`
- 主要覆盖：
  - `get_video_tools`
- 主要观察：
  - 工具状态读取是否正常
  - 是否能自然翻译成用户可理解的人话
  - 是否泄漏内部结构

### 5. 笔记工作流

- 问题：
  - `帮我把这节 API Key 视频的重点记到笔记里，然后把笔记打开。`
- 主要覆盖：
  - `get_video_summary`
  - `save_note`
  - `open_notes`
- 主要观察：
  - 内容读取和动作执行是否能组合起来
  - `save_note` 参数质量
  - 最终动作完成后的回复是否自然

### 6. video scope 边界

- 问题：
  - `当前这个视频讲的是 API Key，那后面哪一节最可能和它衔接？`
- 主要覆盖：
  - 当前视频 summary
  - 或保守的 `direct_response`
- 主要观察：
  - video 上下文是否不会乱越权到整个 series
  - 回答是否不过于僵硬

## 动作矩阵

动作矩阵适合统一跑成一组，每个 prompt 很短，但覆盖产品里的主要动作工具。

### 1. 打开系列首页

- 问题：
  - `打开系列首页`
- 覆盖：
  - `open_series_home`

### 2. 打开系列概览

- 问题：
  - `打开系列概览`
- 覆盖：
  - `open_series_overview`

### 3. 打开概况

- 问题：
  - `打开概况`
- 覆盖：
  - `open_overview`
- 备注：
  - 如果当前视频概况缺失，可能联动 `generate_overview`

### 4. 打开思维导图

- 问题：
  - `打开思维导图`
- 覆盖：
  - `open_mindmap`
- 备注：
  - 如果导图缺失，可能联动 `generate_mindmap`

### 5. 打开知识卡片

- 问题：
  - `打开知识卡片`
- 覆盖：
  - `open_knowledge_cards`

### 6. 打开视频

- 问题：
  - `打开视频`
- 覆盖：
  - `open_video`

### 7. 生成概况

- 问题：
  - `生成概况`
- 覆盖：
  - `generate_overview`

### 8. 生成导图

- 问题：
  - `生成导图`
- 覆盖：
  - `generate_mindmap`

## 建议人工观察点

- 是否调用了明显不该调用的工具
- 是否出现空回答、半截回答、重复回答
- 是否泄漏了内部字段，如 `tool_name`、`payload`、`selected_tool`
- 复杂问题下是否只是分点复述，没有真正回答用户问题
- video scope 下是否越界胡答其他视频
- 动作类请求是否直达动作，不绕成问答
